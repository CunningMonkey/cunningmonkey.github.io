leveldb--跳表:SkipList

1. 概述

leveldb维护在内存中的表类型为MemTable，其本质是一个SkipList。
其整体可以分为三部分，Iterator、Node、表的插入删除操作。

2. 跳表

跳表是一种链表的变形，通过插入冗余节点来实现log(n)时间复杂度的查找。

2.1 Node

Node类型为跳表中的节点，其定义为：

每个Node的对象都存储着当前列更高的节点的地址，Next(n)获得当前节点之上的第n个节点（即高度为n的节点）。SetNext(int n, Node* x)则是将高度为n处的节点设置为x。

2.2 跳表结构

2.3.1 成员变量

其成员变量都是私有变量，分别有：

  1. std::atomic max_height_，当前跳表的最高的高度（列的长度），最大值为12。
  2. Comparator const compare_，比较器，用于比较key的大小
  3. Arena* const arena_，内存分配器
  4. Node* const head，跳表的头节点
  5. Random rnd_，随机数生成器

2.3.2 私有方法

1.创建新节点， NewNode(Key, height)
使用内存分配器，分配一个高度为height的节点的内存，使用该内存初始化一个节点。

  1. 产生随机高度的数值，RandomHeight()

  1. 判断key是否比节点的key更大
    空指针表示的节点认为key无限大

2.3 Iterator

Iterator定义了对跳表的迭代器操作。
