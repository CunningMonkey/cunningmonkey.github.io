leveldb--写操作:Log

Introduction

在写记录时，为了实现crash recovery（还有事务？不知道leveldb有没有提供对事务的处理）。在将记录实际写入到表之前，要先将记录写入到日志中。

在整个写流程中，会使用到两种Writer，第一种是定义在DBImpl中的Writer，第二种则是定义在leveldb::log命名空间内的Writer，该Writer用于写日志。

第一种

第二种

每个db会维护一个dqueue的队列，记录着需要写的writer，是第一种Writer。除此之外还有一个tmp_batch_用来表示一个空的batch，本身没有意义，应该是为了方便代码的书写。还有一个则是log_，是第二种Writer。

之后的MakeRoomForWrite涉及到对表的处理，后续再看。
updates是WriteBatch的指针，记录着要写的操作。

2.2.2 BuildBatchGroup

调用BuildBatchGroup对writers中保存的writer进行打包处理（本质上是打包batch，但是DBImpl::Writer类中会保存一些如sync的状态），同样返回一个WriteBatch的指针，该batch保存了打包后的操作。
BuildBatchGroup实现的是将writers队列中的多个batch打包成一个batch组（本质上也是一个batch）。会把传入的last_writer的地址修改为batch组的最后一个batch的指针的地址。

首先会计算得到一个max_size用来表示batch组的最大size。
使用result记录最终的batch组，然后循环从队列中读writer，将writer中的操作append到result中。

2.2.3 设置sequence number

然后会对打包得到的batch设置属于其的sequence number(可以看做batch中按序记录的所有操作的起始sequence number)

每个数据库DBImpl对象会维护一个sequence number，每次写之前会将sequence number记录在WriteBatch中，每个batch中的操作又有一个count用来记录操作数。
在写完每个batch后db会将sequence number加上该batch的count，用来表示下一个batch。

2.2. 写日志的操作

2.2.1. log::Writer

再回头看一下log::Writer的定义，成员变量有三个:

  1. WritableFile* dest_
    • 写文件，将日志写入到目的文件中进行存储
    • 跟具体操作系统的写文件操作相关，WritableFile是一个接口
  2. int block_offset_
    • 数据会按块写入到文件中，每个块大小为32Byte
    • static const int kBlockSize = 32768;
  3. uint32_t type_crc_[kMaxRecordType + 1];
    • 校验码，写到每个块的前4字节
    • Header is checksum (4 bytes), length (2 bytes), type (1 byte).
    • static const int kHeaderSize = 4 + 2 + 1;

还有一个私有成员函数Status EmitPhysicalRecord(RecordType type, const char* ptr, size_t length)负责将块写入到日志文件中，每次写一个块。

在每次写新的batch时，EmitPhysicalRecord会先写入header，包括4字节校验位、2字节数据长度位，1字节的RecordType位

2.2.1. AddRecord

写日志通过log::Writer的AddRecord方法完成。
传入AddRecord的参数slice就是之前打包好的batch的内容，由于该内容的尺寸可能会大于单个块，所以要对其进行拆分，拆分成多个块，对每个块调用EmitPhysicalRecord方法写入到日志文件中。

当将batch的内容写入到多个连续的块中后，我们需要区分块的头、尾以及中间块。使用RecordType进行区分。

2.2.2 EmitPhysicalRecord

EmitPhysicalRecord负责将batch写入到块中，每次写一个块

每次写会从上次写的结束位置开始（该位置由block_offet维护，而不是直接写一个新的块，但是如果当前块的剩余空间不足以存储header，则会开辟一个新的块存储。

2.3. 写完日志后的操作

写完日志后就可以将写操作添加到内存表中了，会调用InsertInto方法进行该操作（在WriteBatch中进行了介绍）

由于我们每次写的时候是将多个batch打包成一个batch，所以此时writers队列中已经有多个batch完成了写操作，需要将它们全部弹出，注意我们之前使用last_writer记录了最后一个被打包的batch的地址，我们就可以一直pop_front该batch被弹出即可。
