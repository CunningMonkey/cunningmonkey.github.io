任务控制

杀死进程

在命令行中对进程的控制往往使用SIGNAL的形式
Ctrl-C往往用来发出SIGINT的信号，中断当前进程的执行
可以在程序中捕获该信号，自定义处理

此时会忽略Ctrl-C，可以使用Ctrl-/的方式发出SIGQUIT信号，来结束进程

更常用的结束进程的信号是SIGTERM，就是我们熟悉的kill命令

暂停和后台运行进程

SIGSTOP信号用来暂停进程，终端输入为Ctrl-Z
进程被暂停后，可以使用fg、bg来在前台或者后台继续运行
jobs可以查看当前终端相关的进程

可以使用jobs展示的进程的标号来指定进程

还可以发送SIGKILL信号来结束进程，该信号无法被进程捕获，而是会直接杀死进程，但是可能会导致其子进程变成孤儿进程

终端复用-tmux

Session包含多个Window，window包含多个pane

我的配置

Sessions

a session is an independent workspace with one or more windows

tmux 创建一个新的session

tmux new -s NAME 创建一个名称为NAME的session

tmux ls 查看当前所有的session

<C-b> d 推出当前的session（不会销毁）

tmux a 连接上一个session. 使用 -t 可以指定具体的session

Windows

Equivalent to tabs in editors or browsers, they are visually separate parts of the same session

<C-b> c 创建一个window.

<C-d> 关闭当前的window

<C-b> N 前往第N个window

<C-b> p 前往上一个window

<C-b> n 前往下一个window

<C-b> , 更改当前的window的名称

<C-b> w 列出当前所有的window

Panes

Like vim splits, panes let you have multiple shells in the same visual display.

<C-b> " 水平分割当前的pane

<C-b> % 垂直分割当前的pane

<C-b> <direction> 在pane之间导航

<C-b> z 缩放当前的pane

<C-b> [ Start scrollback. You can then press to start a selection and to copy that selection.

<C-b> <space> Cycle through pane arrangements.

端口转发

分为两种情形：

本地端口转发

将远程服务器上某个端口的流量转发到本地的端口中，可以直接通过访问本地端口的形式访问远程端口。

如假设您希望访问远程服务器上的Web应用程序，并且该应用程序在远程服务器的端口 8080 上运行。您可以通过本地端口转发将远程服务器的端口转发到本地计算机的某个端口，以便在本地浏览器中访问该应用程序。

这将建立一个 SSH 连接到远程服务器，并将本地计算机的端口 8888 与远程服务器的端口 8080 绑定。现在，您可以在本地浏览器中访问 http://localhost:8888，实际上访问的是远程服务器上的应用程序。

远程端口转发

将本地机器的某个端口的流量转发到远程服务器的某个端口，其他用户可以通过访问远程服务器的该端口来访问本地机器

如：
假设您希望将本地计算机上的 Web 服务器（运行在端口 80）暴露给远程服务器上的用户。您可以使用远程端口转发将本地计算机的端口转发到远程服务器的某个端口，让远程用户可以通过远程服务器访问您的 Web 服务器。

这将建立一个 SSH 连接到远程服务器，并将远程服务器的端口 8888 与本地计算机的端口 80 绑定。现在，远程服务器上的用户可以通过访问 http://remote-server:8888 来访问您本地计算机上运行的 Web 服务器。

可以通过配置.ssh/config来配置一个端口转发的功能

然后使用ssh vm连接，就会连接到远程服务器，并且将远程的9999端口转发到本地的8888端口
如果不需要打开ssh连接界面，只是希望开启转发功能，可以使用ssh -N vm
